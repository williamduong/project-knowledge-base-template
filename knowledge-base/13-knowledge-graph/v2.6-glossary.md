# Terminology Registry Glossary

Reference guide for the 10 canonical SaaS entities and their aliases.

## Canonical Entities

### 1. Tenant (repo_origin: billing)

**Aliases:** org, workspace, account

**Definition:** Multi-tenant boundary in SaaS platform. Isolates data, subscriptions, and policies.

**Example:**
```json
{
  "canonical_name": "Tenant",
  "id": "org-123",
  "repo_origin": "billing",
  "aliases": ["acme-corp"]
}
```

**When to use:**
- Billing statements, subscription management
- Multi-tenant scope qualification
- Org-wide policies

---

## Programmatic Lookup

### JavaScript

```javascript
const { resolveTerminology } = require('src/lib/ontology');

// Case-insensitive resolution
const entity1 = resolveTerminology('subscription');     // ✅ returns Subscription
const entity2 = resolveTerminology('PLAN');             // ✅ returns Subscription (via alias)
const entity3 = resolveTerminology('UnknownTerm');      // ❌ returns null

// Result object
{
  canonical_name: 'Subscription',
  aliases: ['plan', 'tier', 'license'],
  repo_origin: 'billing',
  microsoft_cdm_mapping: 'Subscription'
}
```

## Common Mistakes

### ❌ Using alias in canonical_name field

```json
{
  "canonical_name": "plan",    // Wrong! Use "Subscription"
  "repo_origin": "billing"
}
```

### ❌ Spelling inconsistency

```json
{
  "canonical_name": "intent",  // Wrong! Case-sensitive in schema
  "repo_origin": "infrastructure"
}
```

### ✅ Correct usage

```json
{
  "canonical_name": "Intent",
  "repo_origin": "infrastructure"
}
```

## Extensibility (Future)

Phase 3+ may add more entities. Rules for extension:

1. **New canonical_name** must be distinct (no collision with existing names or aliases)
2. **Aliases** must not collide with any existing alias or canonical_name
3. **repo_origin** must be one of: billing, auth, gateway, infrastructure (or new subsystems added in governance)
4. **Microsoft CDM mapping** required for cross-platform interoperability
5. **Backward compatibility**: Existing intents must remain valid after registry extends
