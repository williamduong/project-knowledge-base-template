# Source Extraction Task

You are extracting knowledge from source code into a KB documentation file.
This prompt is generated by `kbx extract` — do NOT edit it manually.

## Source file

`{SOURCE_PATH}`

```
{SOURCE_CONTENT}
```

## Target doc path

`{TARGET_DOC_PATH}`

## Existing doc content (if any)

{EXISTING_DOC_CONTENT}

## Instructions

1. **Extract** the following from the source code (only what is visible):
   - Architecture and design patterns used
   - Responsibilities of each exported function/class/module
   - Dependencies (imports, external services, env vars)
   - Known risks, error handling, edge cases
   - Gotchas and non-obvious behaviour

2. **Format** the output as a KB documentation file with YAML frontmatter:
   ```yaml
   ---
   doc_id: <derive from target doc path>
   title: <concise title>
   verification: code-verified
   time_state: current
   extraction_sources:
     - path: {SOURCE_PATH}
       hash: <sha1 of source content, first 12 chars>
       extracted_at: <today's date YYYY-MM-DD>
   ---
   ```

3. **Keep output under 400 lines**. Prefer concise bullet lists over prose.

4. **Do NOT hallucinate** — only describe what is directly visible in the source.
   If something is unclear, note it with `<!-- unclear: ... -->`.

5. **Do NOT include** the raw source code in the output doc.

## Model recommendation

This prompt is designed for cheap/fast models:
- GPT-4o-mini
- Gemini 1.5 Flash
- Claude Haiku 3

Reasoning models (o1, o3) are NOT needed and waste tokens.

## Output

Respond with the complete KB doc markdown (frontmatter + content).
Do not include any preamble or explanation — output the doc only.
