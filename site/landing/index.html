How it works

A CLI Backbone With An Agent Operating Model

The CLI bootstraps and validates the knowledge base locally. The AI workflow stays disciplined because the repository already contains the operating manual, prompt pack, revision state, and governance rules the agent must read first.

CLI bootstrap
Install & initialize
# One-off
npx @williamduong/kb help

# Global install
npm install -g @williamduong/kb
kb help

# Init a new KB
kb init --mode private-git
  • init bootstraps a new KB workspace.
  • sync and update pull forward template changes safely.
  • doctor --strict verifies that the KB is still coherent.
Agent behavior

What the agent reads before touching the project

Start from the repository index and operating rules, not from guesswork. The agent reads the KB map, prompting guide, metadata schema, and revision state before making broad edits.

  • Read the operating manual to understand the maintenance loop and guardrails.
  • Pick a prompt from Prompt Pack instead of inventing an ad-hoc workflow.
  • Check revision state against git before trusting stale documentation.

Get started

Getting Started In 3 Steps

Install the CLI, choose the prompt flow inside Docs, then move straight into implementation.

Step 1 | Install + bootstrap
# Install or run one-off
npm install -g @williamduong/kb
kb help

# Bootstrap a new knowledge base
kb init --mode private-git

# Check health after your first fill pass
kb doctor --strict
  1. Step 1. Install + bootstrap: install the CLI, run kb help, then initialize with kb init --mode private-git.
  2. Step 2. Choose a prompt flow: open the Prompt Library in Docs and select the build, maintenance, release, or patch workflow you need.
  3. Step 3. Validate and start development: run kb doctor, then continue implementation with the KB and finalization plan open.
