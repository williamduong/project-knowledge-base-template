CLI Backbone + Agent Operating Model

The CLI manages the KB lifecycle. The agent stays disciplined because the KB already contains the operating manual, prompt pack, and governance rules it must read first.

CLI Backbone

Six commands cover the full KB lifecycle — from first scaffold to ongoing maintenance.

kb init Scaffold a new KB into the current workspace
kb show / hide Expose or retract the KB mount in private-git mode
kb sync Collect drift evidence since baseline; write sync report
kb update Run sync then stamp the new template version state
kb doctor Publish-readiness checks; --strict exits 1 on any WARN
kb test Smoke-test template structure against expected schema
Agent Operating Model

The KB is the agent’s source of truth. The CLI keeps it coherent; the agent reads first, then acts.

  • Read before acting Opens INDEX.md + operating manual on every session start.
  • Prompt Pack Pre-built workflows for each action — no ad-hoc processes invented.
  • Revision-aware Checks baseline stamp to detect stale docs before trusting content.
  • Verify after writing Runs kb doctor and commits KB alongside code in one pass.

What’s coming

Roadmap & Planned Features

The KB template is actively evolving. These are the capabilities currently being researched or prototyped.

Bi-temporal knowledge & GraphDB

Track when facts were known vs. when they were true. Embed a graph layer to map relationships between entities across the KB.

MCP server integration

Expose the KB as a Model Context Protocol server so agents can query structured knowledge directly via tool calls.

Broader verification coverage

Extend verification workflows and doctor checks to cover more project types: mobile apps, data pipelines, embedded systems.

SaaS-specialized variant

A pre-configured KB profile for SaaS products: tenant model, subscription lifecycle, feature flags, billing domain built in.

Multi-app catalog management

A variant designed for teams managing a linked catalog of applications — shared domain model, cross-system relationships, and dependency tracking.
