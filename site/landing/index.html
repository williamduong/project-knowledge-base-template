Tooling

CLI Preview — kb

A local CLI for initializing, maintaining, and health-checking downstream KB instances. No server required.

Install & run
# One-off
npx @williamduong/kb help

# Global install
npm install -g @williamduong/kb
kb help

# Init a new KB
kb init --mode private-git
  • help — list available commands
  • init --mode private-git|tracked — bootstrap a new KB
  • show / hide — toggle private-git visibility
  • sync [--accept-baseline] — pull latest template changes
  • update [--accept-baseline] — apply version patches
  • doctor [--json] [--strict] — validate KB health
  • test — run template self-checks

Get started

Quick Start

  1. Choose your archetype — open 00-start-here/project-scope-matrix.md and mark the project type and optional modules.
  2. Read working defaults — read 00-start-here/how-to-use-this-kb.md to understand the fill order and governance rules.
  3. Stamp the revision — if the repo is under git, record the baseline commit in 00-start-here/repository-revision-state.md.
  4. Start the maintenance queue — open 00-start-here/finalization-plan.md and work through the priority list.
  5. Use the CLI — run kb doctor regularly to catch metadata drift and broken links early.
