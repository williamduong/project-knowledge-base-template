Severity Policy
hard-fail: immediate exit 1, no auto-retry.
warn: command may proceed, but warning is logged and counted.
info: advisory signal only, still recorded for trend analysis.
Promotion rule: repeated warnings for the same gate and scope can be promoted to hard-fail by policy configuration.
HumanGateRequired Triggers
Escalate to human gate when any high-risk trigger is present (examples):
- destructive mutation_class with medium+ risk
- evidence_state = none or conflicting
- cross-tenant or wide target_scope
- backward-compat uncertainty in minor bump
- repeated failed attempts in generative loop
Generative Loop Contract
attempt_0: run selected pipe and required gates if pass -> continue workflow if fail -> classify failure and propose fix attempt_1: apply bounded correction within same mutation_class re-run required gates attempt_2: one final bounded retry with explicit evidence update re-run required gates after attempt_2 failure: escalate = HumanGateRequired emit fallback_or_escalation with deterministic reason constraints: - max attempts: 3 (0,1,2) - no unbounded loops - no mutation class widening without human approval - idempotency required for repeated apply
