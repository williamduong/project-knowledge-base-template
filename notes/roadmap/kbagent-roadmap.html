KBAgent — Implementation roadmap

Lộ trình từ documentation hiện tại đến giao diện quản lý chạy thực · 5 phases · ~8-12 tuần effort

TL;DR — 5 phases
P0 (1-2 ngày): fix gap analysis P0 items (naming, lifecycle states, tách principles/rules) — không có UI, chỉ doc cleanup.
P1 (3-5 ngày): chọn UI shell — VSCode extension vs localhost webapp vs Electron. Build "Hello World" của lựa chọn đó.
P2 (1-2 tuần): bridge layer — REST API hoặc IPC wrap quanh kbx CLI. Mọi command đều có --json flag rồi, parse output là xong.
P3 (2-3 tuần): UI read-only — đọc intents, chaos history, rules, dispatch tuple. Không edit. Giống kbagent-workspace.html mockup nhưng có data thực.
P4 (2-3 tuần): UI interactive — create/update intent, approve, dispatch, retro. Bao gồm Foundation editor save xuống .kb/.
P5 (1-2 tuần): polish — error handling, search, keyboard shortcuts, theme. Package & ship.

Risks & mitigation
CLI breaking changes
Repo đang ở v2.7-beta, target v2.9. Commands có thể đổi signature. Mitigation: pin kbx version trong UI, mark compatibility range.
Schema drift
JSON output format có thể đổi. Mitigation: schema validation ở bridge layer, fail loud khi parse error thay vì silently corrupt.
Spawn overhead
Mỗi command call spawn Node process (~100-300ms). Mitigation: batch reads, file watcher cho realtime updates, daemon mode (nếu repo support).
Concurrent writes
User dùng cả terminal và UI. Mitigation: file watcher detect changes, prompt user merge/reload. KBX-AX004 single write point giúp ở đây.
